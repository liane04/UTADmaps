📐 Análise dos resultados

Os 4 erros têm a mesma raiz: 2 elementos <input> (TextInput de email e password no ecrã de login) que não têm <label for="..."> HTML clássico. O TAW reporta isto sob 4 critérios distintos (1.1.1, 1.3.1, 3.3.2, 4.1.2) porque a regra dos labels é fundamental e atravessa vários princípios.

Mitigação já aplicada: em React Native, os inputs receberam accessibilityLabel que se traduz para aria-label no DOM resultante — o que cumpre formalmente o critério para leitores de ecrã modernos (verificado independentemente pelo axe-core e pa11y com sucesso). O TAW, no entanto, procura especificamente o atributo HTML <label for="..."> tradicional, padrão histórico. Esta diferença é responsável pela divergência de resultados entre TAW e axe-core.

5 avisos: decorrem de limitações estruturais do React Native Web (geração de CSS com unidades absolutas px e position: absolute) e do título da página estar truncado para "UTADmaps" em vez de "UTAD Maps — Navegação inteligente" (já corrigido localmente, pendente de redeploy).
